How should images sit on the page?

This shapes the whole editor architecture. Pick the model that feels most like what you have in mind.

🌅

A — Inline / Floating (OneNote-style)

Images are placed freely on the page — you tap to position them, drag to move. Text wraps around or flows past them. Most like OneNote desktop.

⚠ Very complex to build on Android. Custom drag-and-drop layout engine required.

🌅
🌸
🌿

B — Block-based (Notion-style)

Each element is a block: text block, image block, or image gallery block. Tap + to insert a block anywhere. Images are full-width or grid.

✓ Natural fit for Compose. Predictable, clean layout.

🌅
🌸

C — Rich text with inline images

Extend the existing richeditor-compose editor — images insert inline into the text flow (like a document). Side-by-side image grids supported.

✓ Builds on existing code. Moderate complexity. Good balance of flexibility.
